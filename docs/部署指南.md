# Agent Bench 部署指南

## 一、环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| 操作系统 | Ubuntu 20.04+ | 本指南以 Ubuntu 为例，macOS 可参考原有流程 |
| Python | 3.9+ | `python3 --version` 确认 |
| JDK | 11+ | hvigor 编译需要 `java`，hmsdk 不自带 |
| Node.js | 18+ | OpenCode 运行需要，`node --version` 确认 |
| Git | 2.x+ | 工作区基线管理 |
| 磁盘空间 | ≥ 10GB | SDK、依赖、编译产物 |
| 内存 | ≥ 4GB | 编译 HarmonyOS 项目 |

## 六、配置项目

### 6.1 修改 `config/config.yaml`

```yaml
# 指向本机 hmsdk 命令行工具根目录
deveco_path: "/home/work/hmsdk/command-line-tools"

task_manager:
  max_concurrency: 3
  cloud_base_url: "http://<云测管理台IP>:3000"
```

- `deveco_path`：**必须配置**，指向 hmsdk 的 `command-line-tools` 目录
- `cloud_base_url`：云测管理台地址，改为本机可访问的地址或保持默认

### 6.2 修改 `config/agents.yaml`

检查所有 agent 的 `opencode_proxy` 配置：

- 如本机**没有代理服务**，确保所有 `opencode_proxy` 被注释或删除
- 如需要代理访问外部 LLM API，取消注释并填入正确代理地址

```yaml
# 无代理环境 - 注释掉
# opencode_proxy:
#   http_proxy: "http://127.0.0.1:7890"
#   https_proxy: "http://127.0.0.1:7890"

# 有代理环境 - 填入实际代理地址
opencode_proxy:
  http_proxy: "http://<代理IP>:<端口>"
  https_proxy: "http://<代理IP>:<端口>"
```

同时确认 `model` 字段指向正确的 LLM 模型：

```yaml
model: "minimax-cn-coding-plan/MiniMax-M2.7"
# model: "openai/gpt-5.4"
# model: "bailian-coding-plan/glm-5"
```

## 七、网络与安全组

### 7.1 执行器对外端口

| 端口 | 用途 | 配置方式 |
|------|------|---------|
| 8000 | 执行器服务（云测管理台调用） | 阿里云安全组入方向放通 TCP 8000 |

### 7.2 阿里云安全组配置

1. 登录阿里云控制台 → ECS → 实例 → 安全组
2. 添加入方向规则：

| 协议 | 端口范围 | 授权对象 | 说明 |
|------|---------|---------|------|
| TCP | 8000/8000 | 0.0.0.0/0 或云测管理台 IP | 执行器服务 |
| TCP | 4096/4096 | 127.0.0.1/32 | OpenCode Server（仅本机访问即可） |

### 7.3 验证端口可达性

```bash
# 启动服务后，从外部机器验证
curl http://<本机公网IP>:8000/api/health

# 预期返回
{"ok":true,"service":"cloud_executor"}
```

## 十、常见问题

### Q1: hvigor 编译报 `spawn java ENOENT`

**原因**：系统未安装 JDK。

```bash
apt-get install -y default-jdk
java --version  # 验证
```

### Q2: 外部访问 8000 端口无响应

**原因**：阿里云安全组未放通。

在 ECS 安全组入方向添加 TCP 8000 规则，授权对象 `0.0.0.0/0`。

### Q3: 连接云测管理台失败

**原因**：本机无法访问 `cloud_base_url` 指定的地址。

检查 `config/config.yaml` 中的 `cloud_base_url` 是否正确，以及本机网络/代理配置。

### Q4: OpenCode 启动后带代理环境变量

**原因**：`agents.yaml` 中某 agent 配置了 `opencode_proxy`，但本机无代理。

注释掉所有 `opencode_proxy` 配置。

### Q5: 编译报 `hvigorw.js 未找到`

**原因**：`config/config.yaml` 的 `deveco_path` 未配置或路径错误。

确保 `deveco_path` 指向正确的 `command-line-tools` 目录，且目录下存在 `hvigor/bin/hvigorw.js`。
