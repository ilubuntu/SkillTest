# Agent Bench 部署指南

## 1. 适用范围

本文档用于部署当前执行器服务。当前产品职责已经收敛为：

- 接收云测任务
- 调用主修复 agent
- 上传产物并上报结果

约束打分和代码打分已不在主流程内。

## 2. 组件关系

部署后实际运行的只有两个组件：

- Executor 服务
- OpenCode Server

其中：

- `deploy.sh` / `deploy.bat` 只负责启动 Executor
- OpenCode 由 Python 程序在启动阶段自动检查并拉起
- HarmonyOS 工具链也由 Python 程序在启动阶段校验

## 3. 环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| 操作系统 | Ubuntu 20.04+ | 云测服务器推荐环境 |
| Python | 3.9+ | 执行器服务运行时 |
| Git | 2.x+ | 工作区基线管理 |
| OpenCode | 最新稳定版 | Agent 执行引擎 |
| HarmonyOS 工具链 | 按平台安装 | 用于预编译和后编译验证 |
| JDK | 11+ | Linux command-line-tools 场景通常需要系统 JDK |
| 磁盘空间 | ≥ 10GB | SDK、依赖、编译产物 |
| 内存 | ≥ 4GB | 编译 HarmonyOS 项目 |

HarmonyOS 工具链支持三类环境：

- macOS: DevEco Studio
- Linux: command-line-tools
- Windows: DevEco Studio

## 4. 云测服务器准备

### 4.1 安装系统依赖

Ubuntu 示例：

```bash
apt-get update
apt-get install -y python3 python3-pip git default-jdk
```

检查：

```bash
python3 --version
java --version
git --version
```

### 4.2 安装 HarmonyOS command-line-tools

云测服务器通常不会安装 DevEco Studio，而是只安装 command-line-tools。假设目录为：

```text
/home/work/hmsdk/command-line-tools/
```

至少需要能解析出：

- `node`
- `hvigor`
- `harmonyos_sdk`
- `java_home`

当前代码会在启动时做硬校验；校验失败会直接拒绝启动。

### 4.3 安装 OpenCode

确保服务器上已经可直接执行：

```bash
which opencode
opencode --version
```

## 5. 配置文件

### 5.1 `config/config.yaml`

当前工具链配置使用平台分组，不再使用单个 `deveco_path`。

```yaml
harmony_toolchain:
  auto_detect: true

  macos:
    node: "/Applications/DevEco-Studio.app/Contents/tools/node/bin/node"
    hvigor: "/Applications/DevEco-Studio.app/Contents/tools/hvigor/bin/hvigorw.js"
    harmonyos_sdk: "/Applications/DevEco-Studio.app/Contents/sdk"
    java_home: "/Applications/DevEco-Studio.app/Contents/jbr/Contents/Home"

  linux:
    node: "/home/work/hmsdk/command-line-tools/tool/node/bin/node"
    hvigor: "/home/work/hmsdk/command-line-tools/hvigor/bin/hvigorw.js"
    harmonyos_sdk: "/home/work/hmsdk/command-line-tools/sdk"
    java_home: "/usr/lib/jvm/java-11-openjdk-amd64"

  windows:
    node: "C:\\Program Files\\Huawei\\DevEco Studio\\tools\\node\\node.exe"
    hvigor: "C:\\Program Files\\Huawei\\DevEco Studio\\tools\\hvigor\\bin\\hvigorw.js"
    harmonyos_sdk: "C:\\Program Files\\Huawei\\DevEco Studio\\sdk"
    java_home: "C:\\Program Files\\Huawei\\DevEco Studio\\jbr"

task_manager:
  max_concurrency: 3
  cloud_base_url: "http://<云测管理台IP>:3000"
```

说明：

- 启动时会优先读取当前平台配置
- 配置不通时，如果 `auto_detect: true`，会自动探测
- 仍无法解析出以下 4 个路径时，服务会直接拒绝启动：
  - `node`
  - `hvigor`
  - `harmonyos_sdk`
  - `java_home`

云测服务器部署时，至少确认：

- `linux.node`
- `linux.hvigor`
- `linux.harmonyos_sdk`
- `linux.java_home`

对应真实可访问路径。

### 5.2 `config/agents.yaml`

确认主修复 agent 的模型配置正确：

- `baseline`
- `harmonyos-plugin`

## 6. 网络与安全组

### 6.1 对外端口

| 端口 | 用途 | 说明 |
|------|------|------|
| 8000 | Executor 服务 | 云测管理台调用 |
| 4096 | OpenCode 服务 | 仅本机访问即可 |

### 6.2 安全组建议

阿里云 ECS 示例：

| 协议 | 端口范围 | 授权对象 | 说明 |
|------|---------|---------|------|
| TCP | 8000/8000 | 云测管理台 IP 或内网网段 | Executor 服务 |
| TCP | 4096/4096 | 127.0.0.1/32 | OpenCode 仅本机访问 |

### 6.3 网络注意事项

- `cloud_base_url` 必须是当前机器可访问的云测管理台地址

## 7. OpenCode 与 Skill 隔离

当前执行器采用：

- 单个共享 OpenCode 服务
- 每个任务独立 `workspace`
- 每个任务独立 `.opencode/skills`

并且执行器启动 OpenCode 时会使用独立的 `XDG_CONFIG_HOME`，避免读取用户全局：

- `~/.config/opencode`

因此：

- 全局 skill 不会自动泄漏到任务里
- 每个任务只会看到当前 agent 配置允许的 skill

## 8. 启动

### macOS / Linux

```bash
./deploy.sh restart
```

### Windows

```bat
deploy.bat restart
```

启动流程：

1. 启动 Executor
2. Executor 启动时检查 OpenCode 可执行文件
3. Executor 启动时校验 HarmonyOS 工具链
4. Executor 启动时自动拉起 OpenCode
5. 服务就绪后监听 8000 端口

说明：

- `deploy.sh` / `deploy.bat` 不再直接启动 OpenCode
- OpenCode 的唯一启动入口在 Python 程序内部

## 9. 健康检查

```bash
curl -s http://127.0.0.1:8000/api/health
curl -s http://127.0.0.1:4096/global/health
```

预期：

- `8000` 返回 Executor 健康结果
- `4096` 返回 OpenCode 健康结果

外部机器还应验证：

```bash
curl http://<服务器IP>:8000/api/health
```

## 10. 云测入口

当前云测入口：

- `POST /api/cloud-api/baseline`
- `POST /api/cloud-api/harmonyos-plugin`
- `GET /api/cloud-api/status`

说明：

- `baseline` 和 `harmonyos-plugin` 分别对应两套主修复 agent 配置
- `status` 用于查看执行状态

云测调用时，应使用服务器可访问地址，例如：

```text
POST http://<服务器IP>:8000/api/cloud-api/baseline
POST http://<服务器IP>:8000/api/cloud-api/harmonyos-plugin
```

## 11. 日志与产物

### 11.1 进程级日志

- `logs/agent_bench.log`

说明：

- 控制台输出与该文件基本同源
- 按小时滚动

### 11.2 单任务日志

每个任务目录下：

- `local_execution.log`

这是单任务的详细文本日志。

### 11.3 云测交互审计

每个任务目录下：

- `cloud_api_events.json`

记录最近一次：

- 进度上报请求/响应
- 最终结果上报请求/响应

### 11.4 典型任务目录

```text
results/execution_<id>_<timestamp>/
├── original/
├── workspace/
├── generate/
├── diff/
├── checks/
├── local_execution.log
└── cloud_api_events.json
```

说明：

- `_download/` 不再保留
- `workspace_meta/` 不再生成
- `workspace_cache/` 仅用于编译隔离，编译结束后自动清理

## 13. 常见问题

### Q1: 启动直接失败，提示 HarmonyOS 工具链检查失败

原因：

- 当前平台配置错误
- 或自动探测失败

处理：

- 先检查 `config/config.yaml` 当前平台配置
- 再确认以下路径实际存在：
  - `node`
  - `hvigor`
  - `harmonyos_sdk`
  - `java_home`

### Q2: Agent 内部编译和执行器预编译结果不一致

当前正确链路是：

- 启动时由 Executor 统一解析工具链
- 统一注入环境变量：
  - `AGENT_BENCH_NODE_BIN`
  - `AGENT_BENCH_HVIGOR_JS`
  - `AGENT_BENCH_HARMONYOS_SDK`
  - `AGENT_BENCH_JAVA_HOME`
- `build-harmony-project` skill 优先使用这组变量

如果不一致，优先检查是否完整重启过服务。

### Q3: OpenCode 看到的 skill 不对

检查两件事：

1. 是否通过 `deploy.sh` / `deploy.bat` 启动，让执行器用独立 `XDG_CONFIG_HOME` 拉起 OpenCode
2. 任务 `workspace` 下是否生成了：
   - `opencode.json`
   - `.opencode/skills/`

### Q4: 查看单个任务日志看哪里

直接看任务目录下：

```bash
tail -f results/execution_<id>_<timestamp>/local_execution.log
```

### Q5: 云测服务器上预编译失败，但本地开发机能编译

优先检查：

- `config/config.yaml` 的 `linux` 配置是否正确
- Linux 机器上是否确实安装了 command-line-tools
- `java_home` 是否指向可用 JDK
- 服务器是否完整重启过，让新工具链配置真正生效
