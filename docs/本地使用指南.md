# 本地使用指南

## 1. 适用场景

用于本地开发机手动启动执行器、触发任务、查看日志和验证 skill 隔离。

## 2. 启动

### macOS / Linux

```bash
./deploy.sh restart
```

### Windows

```bat
deploy.bat restart
```

说明：

- 启动脚本只负责启动 Executor
- OpenCode 由 Python 程序自动检查和拉起
- HarmonyOS 工具链也在启动阶段自动校验

## 3. 健康检查

```bash
curl -s http://127.0.0.1:8000/api/health
curl -s http://127.0.0.1:4096/global/health
```

## 4. 常用接口

### 4.1 本地直接跑已有 case

```bash
curl -s -X POST http://127.0.0.1:8000/api/local/run-case \
  -H 'Content-Type: application/json' \
  -d '{
    "caseDir": "/Users/bb/work/benchmark/github/agent_bench/test_cases/bug_fix/002",
    "agentId": "baseline"
  }'
```

可替换的 `agentId`：

- `baseline`
- `harmonyos-plugin`

### 4.2 查看本地任务状态

```bash
curl -s "http://127.0.0.1:8000/api/local/status?execution_id=<执行ID>"
```

### 4.3 查看云测任务状态

```bash
curl -s "http://127.0.0.1:8000/api/cloud-api/status?execution_id=<执行ID>"
```

## 5. 看日志

### 5.1 进程级日志

```bash
tail -f logs/agent_bench.log
```

用途：

- 看服务启动
- 看主状态
- 看错误摘要

### 5.2 单任务详细日志

```bash
tail -f results/execution_<id>_<timestamp>/local_execution.log
```

用途：

- 看单个任务完整过程
- 看 Agent 详细行为
- 看上传和上报摘要

## 6. 看任务产物

典型目录：

```text
results/execution_<id>_<timestamp>/
├── original/
├── workspace/
├── generate/
├── diff/
├── checks/
├── local_execution.log
└── cloud_api_events.json
```

重点文件：

- `diff/changes.patch`
  - 最终 patch
- `checks/pre_compile_check/compile.log.txt`
  - 预编译日志
- `checks/post_compile_check/compile.log.txt`
  - 后编译日志
- `generate/output.txt`
  - 主修复 agent 输出

## 7. 本地验证 skill 隔离

当前实现是：

- 单个共享 OpenCode 服务
- 每个任务自己的 `workspace/opencode.json`
- 每个任务自己的 `workspace/.opencode/skills`

如果要验证任务实际看到了哪些 skill，最直接的方法是让 agent 在输出里打印当前可见 skill 列表，或者直接检查任务目录下：

- `workspace/opencode.json`
- `workspace/.opencode/skills/`

## 8. 本地排查建议

### 8.1 预编译失败

先看：

- `checks/pre_compile_check/compile.log.txt`

这通常能直接区分：

- 工具链问题
- 工程自身编译问题

### 8.2 Agent 没有实际修改代码

看：

- `diff/changes.patch`
- `generate/output.txt`
- `local_execution.log`

### 8.3 上报云测内容

看：

- `cloud_api_events.json`

这里保留最近一次：

- 进度上报请求/响应
- 最终结果上报请求/响应
