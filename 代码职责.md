# Agent Bench — 代码职责划分

## 总体原则

- 职责单一：每个模块只做一件事
- 依赖单向：执行入口 -> pipeline -> runner
- 通信收口：所有 OpenCode HTTP 交互集中在 `agent_runner/`
- 云端收口：所有云测状态/结果上报集中在 `cloud_api/`

## 当前主链模块

### `agent_bench/executor/`

执行器 HTTP 服务入口。

- `main.py`
  - 启动 FastAPI
  - 检查运行环境
  - 启动或发现 OpenCode 服务
- `cloud_api.py`
  - 提供云测任务入口
  - 提供本地事件和 SSE 查询入口

### `agent_bench/cloud_api/`

云端接口适配层。

- `service.py`
  - 接收任务
  - 编排单任务执行
  - 上报进度和最终结果
  - 上传产物
- `client.py`
  - 统一 HTTP 请求
- `converter.py`
  - 本地结果转云端协议
- `models.py`
  - 云端协议模型

### `agent_bench/pipeline/`

单任务执行主链。

- `case_runner.py`
  - 准备工程
  - 运行主修复 Agent
  - 编译验证
  - 生成 workspace 和 diff 产物
- `compile_checker.py`
  - HarmonyOS 工程编译检查
- `artifacts.py`
  - 本地产物目录和落盘
- `loader.py`
  - 配置、Agent、测试用例加载

### `agent_bench/agent_runner/`

Agent 通信层。

- `adapter.py`
  - AgentAdapter 抽象接口
- `opencode_adapter.py`
  - OpenCode HTTP API 适配器
- `discovery.py`
  - OpenCode 服务发现与自动启动
- `factory.py`
  - 根据 agent 配置创建适配器

## 配置与资源

- `config/agents.yaml`
  - Agent 注册表
- `config/skills/`
  - OpenCode 挂载 skill
- `test_cases/`
  - 测试用例索引和 case 协议
- `uploader/`
  - 上传接口、工厂与具体实现
