## 这次实际遇到的问题

### 1. 缺少 `fastapi` / `uvicorn`

现象：

- 打包环境中缺少执行器运行依赖
- 二进制无法正常启动

处理：

- 在 `.pack_venv` 中安装执行器依赖

### 2. 缺少 `yaml`

现象：

- 二进制运行时报错：
  - `No module named 'yaml'`

原因：

- `loader.py` 运行时依赖 `PyYAML`
- 最开始没有把 `PyYAML` 放进执行器 requirements

处理：

- 在 `agent_bench/executor/requirements.txt` 中增加：
  - `PyYAML>=6.0`
- 重新打包

### 3. 二进制直接运行时，启动体验和脚本不一致

现象：

- 直接运行二进制时，只看到 `uvicorn` 默认日志
- 看不到这些启动提示：
  - 检查依赖
  - 检查 OpenCode
  - 执行器已就绪
  - 日志文件路径

原因：

- 这些逻辑原本只在 `deploy.sh` 里
- 不在二进制入口里

处理：

- 将启动引导逻辑移动到：
  - `agent_bench/executor/main.py`

现在二进制启动时会自己执行：

- 检查依赖
- 检查 Python 依赖
- 启动并检查 OpenCode Server
- 启动执行器服务
- 打印任务入口
- 打印日志文件位置

### 4. OpenCode 自动启动逻辑不可靠

现象：

- 二进制运行时，OpenCode 可能未启动
- 任务执行时才在建会话阶段报错

原因：

- 原来的 `ensure_opencode_server()` 使用了不正确的启动方式
- 非 Windows 下走了：
  - `["nohup", "opencode", "serve", "--port", "4096", "&"]`
- 这不是 `subprocess.Popen()` 的正确用法

处理：

- 改为直接使用：
  - `subprocess.Popen(["opencode", "serve", "--port", "4096"], ...)`
- 使用 `start_new_session=True`
- 启动后轮询：
  - `http://localhost:4096/global/health`

### 5. Agent 会话创建失败后，流程仍然标记完成

现象：

- OpenCode 建会话失败时，日志里仍然继续：
  - 上传输出代码
  - 标记任务执行完成

原因：

- `adapter.execute()` 失败后返回空字符串
- 外层没有把这个失败态抛出来

处理：

- 在 `OpenCodeAdapter` 里记录最近一次错误：
  - `get_last_error_message()`
- 在 `case_runner.py` 中，如果：
  - `output_text` 为空
  - 且 `last_error_message` 非空
- 直接抛异常

随后在 `cloud_api/service.py` 中：

- 如果 `run_single_case()` 没返回成功状态
- 直接转失败，不再继续上传输出代码，不再标记完成

## 当前二进制的启动行为

直接运行：

```bash
./dist/agent-bench-executor/agent-bench-executor
```

当前会自动完成：

1. 检查依赖
2. 检查 Python 依赖
3. 启动 OpenCode Server
4. 检查 OpenCode 是否可用
5. 启动执行器服务
6. 输出就绪提示

日志会写到：

- `logs/agent_bench_YYYYMMDD_HHMMSS.log`

当前日志指针文件：

- `logs/current_executor_log`
