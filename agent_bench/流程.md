# 评测系统运行流程详解

## 总体流程图

```
python3 run_eval.py --suite bug_fix
       │
       ▼
  加载测试用例 (JSON)
  每个用例自带 eval_type + subject
       │
       ▼
  ┌─── 循环每个用例 ───────────────────────────────────────────┐
  │                                                             │
  │  根据 eval_type 加载被测对象:                                │
  │    skill         → 读取 skills/{subject}.md 的文本          │
  │    mcp_tool      → 定位 mcp_configs/{subject}.json 路径     │
  │    system_prompt  → 读取 prompts/{subject}.md 的文本         │
  │                                                             │
  │  ① 基线运行: claude -p "任务+代码"                → 输出A    │
  │  ② 增强运行: 根据 eval_type 构建不同命令           → 输出B    │
  │  ③ 规则评分: 对 A 和 B 做关键词匹配                          │
  │  ④ LLM评分:  claude -p "评分prompt" 对 A/B 打分             │
  │  ⑤ 汇总该用例的分数                                          │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
       │
       ▼
  生成报告 (JSON + Markdown)
```

## 三种评测类型的调用差异

### Skill 类型

```
基线: claude -p "任务描述 + 代码"
增强: claude -p "Skill内容 + 任务描述 + 代码"
                 ↑ Skill 文本直接拼入 prompt
```

### MCP Tool 类型

```
基线: claude -p "任务描述 + 代码"
增强: claude -p "任务描述 + 代码" --mcp-config mcp_configs/xxx.json
                                  ↑ 挂载 MCP Server，Agent 可调用工具
```

### System Prompt 类型

```
基线: claude -p "任务描述 + 代码"
增强: claude -p "任务描述 + 代码" -s "system prompt 内容"
                                  ↑ 注入 system prompt
```

三种类型的 **基线运行完全一致**（纯 Agent 无增强），差异只在增强运行的命令参数。

## 逐步详解

### 第一步：入口 `run_eval.py`

```python
cases = load_test_cases("bug_fix")    # 读取 test_cases/bug_fix/*.json
# 每个 case 自带 eval_type 和 subject
```

扫描 `test_cases/bug_fix/` 目录，加载所有 JSON 用例。用例可以混合不同的 eval_type。

### 第二步：加载被测对象

```python
subject_content, subject_path = load_subject(eval_type, subject_name)
# skill:         → ("markdown内容", "skills/bug_fix.md")
# mcp_tool:      → (None, "mcp_configs/compile.json")
# system_prompt:  → ("prompt内容", "prompts/arkts_expert.md")
```

### 第三步：基线运行（无增强）

`runner/agent_runner.py` 的 `run_baseline()` 被调用：

```
构造 prompt:
  "你是一个ArkTS开发者。请完成以下任务。
   ## 任务
   {description}
   ## 代码
   {input.ets 的内容}
   ## 要求
   只输出修复后的完整代码"

执行:
  subprocess.run(["claude", "-p", prompt])
                    │
                    ▼
         Claude Code 启动一个全新进程
         读取 prompt → LLM 推理 → 输出代码
                    │
                    ▼
         捕获 stdout 作为 baseline_output（输出A）
```

关键点：`claude -p` 是非交互模式，进程启动 → 输出结果 → 退出，没有对话历史。

### 第四步：增强运行

`run_enhanced()` 根据 eval_type 构建不同的命令：

```python
# Skill: Skill 文本拼入 prompt
cmd = ["claude", "-p", "Skill内容 + 任务描述 + 代码"]

# MCP Tool: prompt 不变，加 --mcp-config 参数
cmd = ["claude", "-p", "任务描述 + 代码", "--mcp-config", "mcp_configs/xxx.json"]

# System Prompt: prompt 不变，加 -s 参数
cmd = ["claude", "-p", "任务描述 + 代码", "-s", "system prompt 内容"]
```

**对比本质**：同样的任务、同样的代码，区别只是有没有增强手段。

### 第五步：规则评分

`evaluator/rule_checker.py` 对输出 A 和输出 B 分别做关键词检查：

```
以用例 001 为例:
  must_contain: ["LazyForEach", "cachedCount", "IDataSource"]
  must_not_contain: ["ForEach(this.items"]

对 baseline_output 检查:
  "LazyForEach" in output?    → True  ✓
  "cachedCount" in output?    → True  ✓
  "IDataSource" in output?    → True  ✓
  "ForEach(this.items" not in output? → True  ✓
  → rule_score = 4/4 * 100 = 100.0

对 enhanced_output 做同样的检查
```

规则评分是**确定性的**，不依赖 LLM，跑多少次结果一样。

### 第六步：LLM 评分（LLM-as-Judge）

`evaluator/llm_judge.py` 再次调用 `claude -p`，让 LLM 当评委打分：

```
构造评分 prompt:
  "你是一个严格的代码评审专家...
   ## 原始代码（有bug）
   {input.ets}
   ## 参考答案
   {expected.ets}
   ## 待评估代码
   {baseline_output 或 enhanced_output}
   ## 评分维度
   - correctness（权重40%）: ...
   - completeness（权重30%）: ...
   - code_quality（权重30%）: ...
   请对每个维度打分（0-100），返回JSON"
```

注意：**每个用例要调 4 次 claude -p**：
1. 基线运行（生成代码）
2. 增强运行（生成代码）
3. 基线评分（LLM 打分）
4. 增强评分（LLM 打分）

### 第七步：分数汇总

```python
# 权重: 规则 30% + LLM 70%
total = 0.3 * rule_score + 0.7 * llm_weighted_avg

# llm_weighted_avg 按 rubric 权重计算:
# correctness(40%) * 85 + completeness(30%) * 72 + code_quality(30%) * 78
# = 34 + 21.6 + 23.4 = 79.0

# total = 0.3 * 100 + 0.7 * 79.0 = 30 + 55.3 = 85.3
```

### 第八步：生成报告

`report/reporter.py` 汇总所有用例的结果：

```
输入: 所有用例的分数明细
       │
       ▼
计算: 平均分、通过率、各维度均分、增益值
       │
       ├── report.json  (机器可读，完整数据)
       └── report.md    (人可读，表格化展示)
```

## 一次完整运行的调用链

以 3 个 skill 类型用例为例，总共 **12 次 `claude -p` 调用**：

```
用例001: claude -p (基线) → claude -p (增强) → claude -p (评基线) → claude -p (评增强)
用例002: claude -p (基线) → claude -p (增强) → claude -p (评基线) → claude -p (评增强)
用例003: claude -p (基线) → claude -p (增强) → claude -p (评基线) → claude -p (评增强)
```

每次调用都是独立进程，互不影响，上下文天然隔离。
